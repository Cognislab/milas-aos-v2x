# MiLAS v2.x — Glossary
Cognis-lab / Mameta Edane  
2026

This glossary defines key terms used in the MiLAS (Mind-Layered Architecture System)
framework. It provides consistent terminology for research papers, implementations,
and future extensions.

---

## L1 — Stability Layer
The foundational cognitive layer responsible for maintaining stable internal states.
It provides rebootability, recovery, and baseline cognitive safety.

## L2 — Generation Layer
The cognitive processing engine that generates, transforms, and integrates
cognitive protocols. It handles the production of internal processes.

## L3 — Governance Layer
The meta-control layer that supervises cognitive processes.  
It manages context, goals, and policies to maintain cognitive coherence.

## Cognitive Protocol
A structured unit of cognitive processing that flows through L1/L2/L3.
Protocols define how information is generated, transformed, and governed.

## Cognitive Stability
The property of maintaining a coherent internal state without collapse,
overload, or runaway processes.

## Cognitive Safety
The engineering guarantee that cognitive processes remain within safe,
recoverable bounds across all layers.

## Rebootability
The ability of the cognitive system to reset or recover from unstable states
without losing structural integrity.

## Internal OS (Inner Operating System)
The conceptual model in which the mind is treated as an operating system,
with layered architecture, protocols, and governance.

## Cognitive Coherence
The alignment between generated processes (L2) and governing policies (L3),
ensuring consistent behavior and interpretation.

## AOS — Architecture Operating System
The structural framework that defines how MiLAS layers interact as a unified OS.
